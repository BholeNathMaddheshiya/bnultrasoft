Introduction to Internet of Things

Point to remember:

✬ IoT stands for Internet of Things.

✬ The term Internet of Things was invented in 1999 by the computer scientist Kevin Ashton.

✬ IoT, is a network of physical devices. These devices can transfer data to one another without human beings.

✬ The main goal of the IoT is to create self-reporting devices that can communicate with each other in real time.

✬ The IoT device includes such as smart coffee makers, smart washing machine, smart home security systems like fire alarm , smart webcams , smart Apple Watches, routers, smart speaker devices like Amazon Alexa and Google Home.


How Does IoT Works?


✬ Step 1: IoT devices are physical devices like sensors, that collect data.

✬ Step 2: When the data is collected by the sensors then it shared to an IoT gateway, which acts as a central hub.

✬ Step 3: The software then analyzes the data and transmits the data to users via an app or website.


Advantage of IoT

  • 1). We can easily access any information setting from any part of the globe in real time even if we are not physically present.
  • 2) It minimizes human effort because IoT devices connect and communicate with one another and perform a variety of tasks without the need for human beings.
  • 3) Patient care can be performed more effectively in real time without the need for a doctor’s visit.
  • 4) By automating activities, it saves us a lot of time, it reduces the human effort then it definitely saves out time.

Disadvantage of IoT

  • 1). People are most habituated to have a click based work making them lazy to any sort of physical activities.
  • 2). We lose control of our lives—our lives will be fully controlled, reliant and dependent on technology.
  • 3). Unskilled workers are at a high risk of losing their jobs, which could lead to unemployment.
  • 4). Even we are provide higher security, hackers may access to the system and steal personal information. Due to so many devices to the internet, there is a risk that our information as it can be misused.

Characteristics of IoT


  • 1). Connectivity :- Connectivity is an essential feature of IoT. IoT can connect through several wireless devices, like sensors, mobile phones, trackers, etc. This way, the person will not have to wait for an internet connection to operate a device.
  • 2). Identity of Things :- Identity helps to differentiate between various internet devices. It is essential to give a unique identity to every device so that we can select the device and send the command to it.
  • 3). Intelligence :- The intelligence of IoT devices depends on the sensors' intelligence. As we know the sensors are automatically detects the changing environment send the data to the user for further analysis.
  • 4). Dynamic and Self-Adapting :- IoT devices are dynamic, Assume a CCTV camera meant for surveillance. It should be work in different conditions and different light situations (morning, afternoon, and night).
  • 5). Scalability :- The number of elements connected to the IoT zone is increasing day by day. Hence, an IoT setup can capable of handling the massive expansion. it can handled appropriately and accurately.
  • 6). Self Upgradation :- It also allows the set up of a network for the addition of any new IoT devices. Thus, the technology can quickly start working without delay if the setup has already been done.
  • 7). Architecture :- The architecture of IoT is designed in a way that it is capable of supporting various devices, technologies, and protocols. Its main work is to confirm whether each connected device does not interfere with the other.
  • 8). Security :- With the increasing number of IoT devices, issues regarding the security of personal data have arisen. There might be a chance of data leakage, Maintaining safety and security is a big dare for IoT. However, it still handles it without any disruption.
  • 9). Network :- With the increasing number of IoT devices in a network, it becomes difficult to maintain communication for proper functioning. However, cloud service and gateway are a few methods that can solve such problems. Because IoT devices can communicate with one another, it is more effective and adaptable than other current technologies.
  • 10). Data :- The data gathered from IoT devices are analyzed for future prediction. Example weather condition information which is collected by sensor (Temperature and humidity sensors).

Main Components of IoT

✬ There are typically 4 main components of IoT :-

  • ✬ Sensors and actuators
  • ✬ Connectivity
  • ✬ Data processing
  • ✬ User interface

✬ Sensors and actuators :-
These are the devices that collect data from the environment and carry out actions based on that data. They are sometimes referred to as “things” in IoT.
Sensors are used to collect data, such as temperature, light, sound, or pressure. Actuators, on the other hand, are used to carry out an action based on the data that has been collected, such as turning on a light or opening a door.

✬ Connectivity :- Connectivity element plays a very significant task in IoT. After the data is collected, it needs a way to get to the cloud (or other data storage location) so it can be processed and analyzed. This is where connectivity comes in.
Connectivity refers to the various technologies that are used to connect devices to the internet, such as WiFi, Bluetooth, cellular, or satellite. The data that was gathered through the sensors is then transmitted over the internet using one of these technologies.

✬ Data processing :- After data successfully uploads to the cloud, computing software, and big-data analytics process it needs to be processed and analyzed through data processing for the end-user. For example:- night vision and motion sensors can detect unauthorised activity within a defined parameter. Still, the signals get sent to the user via an alert or video footage.

✬ User interface :- Last but not least, we have the user interface. The user interface is what allows humans to interact with IoT devices and systems. This is the last stage of the data processing. After getting notifications through email or text message, it helping us make a decision, the user may also be able to perform an action and affect the system, it also automatically notify security teams to take action on it.
