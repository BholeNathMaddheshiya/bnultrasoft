Introduction to Internet of Things

Point to remember:

✬ IoT stands for Internet of Things.

✬ The term Internet of Things was invented in 1999 by the computer scientist Kevin Ashton.

✬ IoT, is a network of physical devices. These devices can transfer data to one another without human beings.

✬ The main goal of the IoT is to create self-reporting devices that can communicate with each other in real time.

✬ The IoT device includes such as smart coffee makers, smart washing machine, smart home security systems like fire alarm , smart webcams , smart Apple Watches, routers, smart speaker devices like Amazon Alexa and Google Home.


How Does IoT Works?


✬ Step 1: IoT devices are physical devices like sensors, that collect data.

✬ Step 2: When the data is collected by the sensors then it shared to an IoT gateway, which acts as a central hub.

✬ Step 3: The software then analyzes the data and transmits the data to users via an app or website.


Advantage of IoT

  • 1). We can easily access any information setting from any part of the globe in real time even if we are not physically present.
  • 2) It minimizes human effort because IoT devices connect and communicate with one another and perform a variety of tasks without the need for human beings.
  • 3) Patient care can be performed more effectively in real time without the need for a doctor’s visit.
  • 4) By automating activities, it saves us a lot of time, it reduces the human effort then it definitely saves out time.

Disadvantage of IoT

  • 1). People are most habituated to have a click based work making them lazy to any sort of physical activities.
  • 2). We lose control of our lives—our lives will be fully controlled, reliant and dependent on technology.
  • 3). Unskilled workers are at a high risk of losing their jobs, which could lead to unemployment.
  • 4). Even we are provide higher security, hackers may access to the system and steal personal information. Due to so many devices to the internet, there is a risk that our information as it can be misused.

Characteristics of IoT


  • 1). Connectivity :- Connectivity is an essential feature of IoT. IoT can connect through several wireless devices, like sensors, mobile phones, trackers, etc. This way, the person will not have to wait for an internet connection to operate a device.
  • 2). Identity of Things :- Identity helps to differentiate between various internet devices. It is essential to give a unique identity to every device so that we can select the device and send the command to it.
  • 3). Intelligence :- The intelligence of IoT devices depends on the sensors' intelligence. As we know the sensors are automatically detects the changing environment send the data to the user for further analysis.
  • 4). Dynamic and Self-Adapting :- IoT devices are dynamic, Assume a CCTV camera meant for surveillance. It should be work in different conditions and different light situations (morning, afternoon, and night).

  • 6). Self Upgradation :- It also allows the set up of a network for the addition of any new IoT devices. Thus, the technology can quickly start working without delay if the setup has already been done.
  • 7). Architecture :- The architecture of IoT is designed in a way that it is capable of supporting various devices, technologies, and protocols. Its main work is to confirm whether each connected device does not interfere with the other.
  • 8). Security :- With the increasing number of IoT devices, issues regarding the security of personal data have arisen. There might be a chance of data leakage, Maintaining safety and security is a big dare for IoT. However, it still handles it without any disruption.
  • 9). Network :- With the increasing number of IoT devices in a network, it becomes difficult to maintain communication for proper functioning. However, cloud service and gateway are a few methods that can solve such problems. Because IoT devices can communicate with one another, it is more effective and adaptable than other current technologies.
  • 10). Data :- The data gathered from IoT devices are analyzed for future prediction. Example weather condition information which is collected by sensor (Temperature and humidity sensors).

Main Components of IoT

✬ There are typically 4 main components of IoT :-

  • ✬ Sensors and actuators
  • ✬ Connectivity
  • ✬ Data processing
  • ✬ User interface

✬ Sensors and actuators :-
These are the devices that collect data from the environment and carry out actions based on that data. They are sometimes referred to as “things” in IoT.
Sensors are used to collect data, such as temperature, light, sound, or pressure. Actuators, on the other hand, are used to carry out an action based on the data that has been collected, such as turning on a light or opening a door.

✬ Connectivity :- Connectivity element plays a very significant task in IoT. After the data is collected, it needs a way to get to the cloud (or other data storage location) so it can be processed and analyzed. This is where connectivity comes in.
Connectivity refers to the various technologies that are used to connect devices to the internet, such as WiFi, Bluetooth, cellular, or satellite. The data that was gathered through the sensors is then transmitted over the internet using one of these technologies.

✬ Data processing :- After data successfully uploads to the cloud, computing software, and big-data analytics process it needs to be processed and analyzed through data processing for the end-user. For example:- night vision and motion sensors can detect unauthorised activity within a defined parameter. Still, the signals get sent to the user via an alert or video footage.

✬ User interface :- Last but not least, we have the user interface. The user interface is what allows humans to interact with IoT devices and systems. This is the last stage of the data processing. After getting notifications through email or text message, it helping us make a decision, the user may also be able to perform an action and affect the system, it also automatically notify security teams to take action on it.


Layers of IoT



✬ Perception Layer : This is the first layer of IoT architecture. In the perception layer, number of sensors and actuators are used to gather useful information like temperature, moisture content, intruder detection, sounds, etc. The main function of this layer is to get information from surroundings and to pass data to another layer so that some actions can be done based on that information.

✬ Network Layer : As the name suggests, it is the connecting layer between perception and middleware layer. It gets data from perception layer and passes data to middleware layer using networking technologies like 3G, 4G, UTMS, WiFI, infrared, etc. This is also called communication layer because it is responsible for communication between perception and middleware layer. All the transfer of data done securely keeping the obtained data confidential.

✬ Middleware Layer : Middleware Layer has some advanced features like storage, computation, processing, action taking capabilities. It stores all data-set and based on the device address and name it gives appropriate data to that device. It can also take decisions based on calculations done on data-set obtained from sensors.

✬ Application Layer : The application layer manages all application process based on information obtained from middleware layer. This application involves sending emails, activating alarm, security system, turn on or off a device, smartwatch, smart agriculture, etc.

✬ Business Layer : The success of any device does not depend only on technologies used in it but also how it is being delivered to its consumers. Business layer does these tasks for the device. It involves making flowcharts, graphs, analysis of results, and how device can be improved, etc.


IoT level

✬ An IoT ecosystem can be designed on different levels. Each level is fundamentally a roadmap of how to deploy the components of the IoT ecosystem.

There are six different levels of IoT :-

1). Level 1:

✬ In this level all the components are deployed locally. There is no any cloud or external network involved in the process.

✬ The data sensed in stored locally.

✬ The data analysis is done locally.

✬ The data generated in this level application is not huge.

✬ Example: Room temperature is monitored using temperature sensor and data is stored/analysed locally. Based on analysis made, control action is triggered using mobile app or it can just help in status monitoring.


Diagram of IoT Level 1






2). Level 2:

✬ In this level all the components are deployed locally except the servers. There is a cloud or an external network involved in the process.

✬ At the user’s end there are only sensors, routers and application. The cloud part is having the servers configured for the purpose of storage. This standard is good for those ecosystems where data is big. A huge amount of data is coming from many components in the ecosystem and that too on a rapid speed.

✬ Data analysis is carried out locally. Cloud is used for only storage purpose.

✬ An example of Level 2 IoT is a smart factory (product details such as product name, packing date, expring date etc. ).


Diagram of IoT Level 2






3). Level 3:

✬ In this level all the components are deployed locally, but the servers, data process and data analysed part are done on cloud.

✬ Here, also data is big. Frequency of data sensing is fast and collected sensed data is stored on cloud.

✬ Data analysis is done on the cloud side and based on analysis control action is triggered using mobile app or web app.

✬ An example of Level 3 IoT is a smart industry (tracking/monitoring of packing, services etc ).


Diagram of IoT Level 3






4). Level 4:

✬ At this level-4, multiple sensors are used which are independent of the others which are deployed locally.

✬ Here, the observer node is present on both side, which can observed/monitored the changing data or what happening in real time.

✬ The data collected using these sensors are uploaded to the cloud separately. The cloud storage is used in this level due to requirement of huge data storage.

✬ The data analysis is performed on the cloud and based on which control action is triggered either using web app or mobile app.

✬ An example of Level 4 IoT is a smart cities (noise monitoring, pollution observing etc ).


Diagram of IoT Level 4






5). Level 5:

✬ At this level-5, Here, Multiple device are connected to the sensors and the sensing data send to the coordinator node.

✬ The coordinator node collects data from the multiple device in locally then it send to the cloud.

✬ The data store and data analysis is performed on the cloud and based on which control action is triggered either using web app or mobile app.

✬ There is an Observer Node also in the cloud part that observes the entire process.


Diagram of IoT Level 5






6). Level 6:

✬ At this level-6, Here, Multiple device are connected to the sensors and the sensing data send to the Centralized Controller. .

✬ Here, the observer node is present on both side, which can observed/monitored the changing data or what happening in real time.

✬ The Centralized Controller is present in cloud. But Centralized Controller monitors the status of all multiple device and send control commands to all connected devices to take an action.

✬ The data is store and data analysis is performed on the cloud and based on which control action is triggered either using web app or mobile app.

✬ There is an Observer Node also in the cloud part that observes the entire process.

IoT design Methodology

✬ IoT design methodology includes the following steps:-

  • Step 1 : The purpose and requirements of the system:- Here, We determine the purpose and requirements of the IoT ecosystem for example if it is for home automation, the purpose is to be able to operate home appliances anytime from anywhere in the world. Accordingly, the requirements are like necessary sensors, actuators, security systems, routers, UI etc.
  • Step 2 : The process specification:- This step is about determining the use cases of different IoT systems based on the purpose and the requirements.
  • Step 3 : The domain models:- It clarifies about the category of the IoT system that is proposed. For example, if we want to be able to track temperature of a patient, the IoT system is of healthcare domain.
  • Step 4 : The information models:- It defines the structure of all the information of data that will be produced in the IoT ecosystem.
  • Step 5 : The service specifications:- It will clarify all the services such as inputs, possible outputs and other data-store related information.
  • Step 6 : The levels of IoT:- Depending upon the domain model and the objective, a certain level of IoT is determined in this step. It can be any level between level 1 and level 6.
  • Step 7 : The functional view:- It determines all the functionalities the IoT ecosystem will offer many functional groups are designed to specify which functionality of the system falls under which functional group.
  • Step 8 : The operational view:- It determines the operational part of IoT ecosystem. It can include all the operations like setting up network, deployment of servers, development of UI etc.
  • Step 9 : Integration of devices and components:- It this step is about organizing all the components together in order to form a complete IoT system it can include the sensors, actuators, communication systems, network systems etc.
  • Step 10 : Development of IoT application:- This is the final step, this is the process of designing, building, and implementing software application.

The Physical Design of IoT

✬ A physical design of an IoT system refers to the Things in IoT ( individual node devices ) and their protocols that are utilized to create a functional IoT ecosystem. Here, we deal with the actual components, protocols and things.



Things in IoT

  • ✬ Things in IoT: Things refer to all the components that relate with each other to perform a certain task.
  • 1) Connectivity: Devices like USB hosts and ETHERNET are used for connectivity between the devices and the server.
  • 2) Processor: A processor like a CPU and other units are used to process the data. These data are further used to improve the decision quality of an IoT system.
  • 3) Audio/Video Interfaces: An interface like HDMI (High-Definition Multimedia Interface) devices is used to record audio and videos in a system.
  • 4) Input/Output interface: To give input and output signals to sensors, and actuators.
  • 5) Storage Interfaces: Things like Hard Disk are used to store the data generated from an IoT device.
  • 6) Graphics: GPU (graphics processing unit) are used to accelerate the creation of images.
  • 7) Memory Interfaces : Memory like RAM (Random Access Memory) and Register are used for immediate data storage and retrieval.

IoT Protocols

  • ✬ IoT Protocols: IoT protocols help send commands and data between a network of devices controlled by sensors or other physical attributes like motion, temperature, or vibration. Here we categorize them on TCP/IP layers:

    1). Application
    2). Transport
    3). Network
    4). Link layer
  • 1) Application Layer : This protocols define how the data can be sent over the network using the application interface.
  • Hyper Text Transfer Protocol (HTTP) : It provides a standard way to transfer documents and their contents in the form of HTML pages.
  • Constrained Application Protocol (COAP) : It is server-based protocol. With this protocol, the COAP packet can be shared between different client nodes which are commanded by the COAP server.
  • WebSocket: WebSocket is bidirectional, full-duplex protocol that is used to providing simultaneous two-way communication channels over a single Transmission Control Protocol (TCP) connection.
  • Message Queuing Telemetry Transport (MQTT) :It is a machine to machine (M2M) connectivity protocol, this protocol is based on the publish-subscribe methodology in which clients receive the information through a broker only to the subscribed topic.
  • Extensible Messaging Presence Protocol (XMPP): It provides instant messaging framework, allowing devices to exchange data, realtime status updates, Push Notifications. This protocol is mostly used by instant messaging applications like WhatsApp.
  • Data Distribution Service (DDS) : DDS is another publish-subscribe protocol but it is different from MQTT which connects them to server but here DDS protocol is a broker-less architecture that’s why it is a high speed and high-performance protocol than MQTT as it is not dependent upon any intermediary system.
  • Advanced Message Queuing Protocol (AMQP) : Here, Queues as a result it is focused on not losing messages. It useful for sending transactional messages that’s why mainly used in financial industry.

  • 2) Transport Layer : Protocols of this layer work for the better transfer reachability of data packets along the network.
  • Transmission Control Protocol ( TCP ) : TCP ensures reliable, ordered data delivery by establishing a connection, performing error correction, and retransmitting lost packets.
  • User Datagram Protocol ( UDP ) : UDP is a connectionless protocol that prioritizes speed and low overhead but does not guarantee reliable delivery or perform error correction.

  • 3) Network Layer : Protocols of this layer work for the better routing and forwarding of data packets along the network .
  • Internet Protocol version 4 ( IPv4 ) : Whenever a device is connected to internet, it assigned a 32-bit unique address which identifies a device on a network and makes it possible to connect our devices in IoT ecosystem.
  • Internet Protocol version 6 ( IPv6 ) : It is Latest version of Internet Protocol which is 128-bit addresse , IPv6 includes built-in security features and QoS support, which were not present in IPv4. IPv6 also provides better support for mobile networks and IoT devices, which require large address spaces.
  • IPv6 over Low-Power Wireless Personal Area Networks ( 6LowPAN ) : Internet Protocol uses to the smallest devices, enabling low-power devices with limited processing capabilities to participate in the Internet of Things.

  • 4) Link Layer : Link-layer protocols are used to send data over the network’s physical layer. It also determines how the packets are coded and signaled by the devices.
  • IEEE 802.3 (Ethernet): Ethernet can be used to connect fixed wired IOT devices to produce high speed local area network.
  • IEEE 802.11 (Wi-Fi): Wifi is used in most home and office networks to allow laptops, printers, smartphones, and other devices to communicate with each other.
  • IEEE 802.16 (Wi-Max): Worldwide Interoperability for Microwave Access (WiMAX), is a telecommunications technology aimed at providing wireless data over long distances in a variety of ways, from point-to-point links to full mobile cellular type access.
  • IEEE 802.15.4 (Zigbee/LR-WPAN): Zigbee is a protocol used to link smart devices like lights, plugs, and smart locks to a home network. Low-Rate Wireless Personal Area Network (LR-WPAN)
  • 2G/3G/4G/5G: It is different cellular telephony generations.

2). Communication
3). Security

  • ✬ Communication: Handles the communication for the IoT system.

  • ✬ Management: This blocks provides various functions to manage the IoT system.
  • ✬ Security: This block secures the IoT system and by providing functions such as authentication , authorization, message and data security.
  • ✬ Application: This is an interface that the users can use to control and monitor various aspects of the IoT system. Application also allow users to view the system status and view or analyze the processed data.


✬ IoT communications models

In IoT system there are four communication models.

  • 1) Request-Response communication model :
  • 2) Publish-Subscribe communication model:
  • 3) Push-Pull communication model:
  • 4) Exclusive Pair communication model:

1) Request-Response communication model :




2) Publish-Subscribe communication model:

  • ✬ Publishers are the source of data, Publishers are not aware of the consumers..
  • ✬ Publishers send the data (topics wise) which are managed by the broker.
  • ✬ When the broker receive data for a topic from the publisher, it sends the data to all the subscribed consumers.


3) Push-Pull communication model :

  • ✬ Publishers are not aware of the consumers.
  • ✬ Queues help in send the messaging between the Producers and Consumers.
  • ✬ Queues also act as a buffer which helps in situations when there is a mismatch between the rate at which the producers push data and the rate at which the consumer can pull data.


4) Exclusive Pair communication model :

  • ✬ Client and server can send messages to each other after connection setup.
  • ✬ Connection is setup it remains open until the client sends a request to close the connection.



IoT Communication APIs

In IoT, there are 2 communication APIs:–

✬ REST Based Communication APIs

✬ Web Socket Based Communication APIs



1). REST Based Communication APIs



  • ✬ It focus on a system’s resources and how resource states are addressed and transferred.
  • ✬ REST APIs follow the request-response communication model.
  • ✬ REST APIs are stateless, meaning that each request is processed independently.


2). Web Socket Based Communication APIs



  • ✬ Web Socket APIs allow bi-directional, full-duplex communication between clients and servers.
  • ✬ It follows the exclusive pair communication model.
  • ✬ This Communication API does not require a new connection to be set up for each message to be sent between clients and servers. Once the connection is set up the messages can be sent and received continuously without any interruption.
